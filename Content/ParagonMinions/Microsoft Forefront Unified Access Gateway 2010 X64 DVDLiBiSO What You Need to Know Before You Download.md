
 
# How to Install Microsoft Forefront Unified Access Gateway 2010 Service Pack 4
 
Microsoft Forefront Unified Access Gateway (UAG) 2010 is a software solution that provides secure remote access to corporate resources. It supports various client devices and applications, such as Windows 8, Windows Phone 8, Office 2013, SharePoint 2013, and Exchange 2013. In this article, we will show you how to install Service Pack 4 (SP4) for Forefront UAG 2010, which includes new features and fixes for various issues.

## Prerequisites
 
Before you install Forefront UAG 2010 SP4, you must have the following prerequisites:
 
- Forefront UAG 2010 Service Pack 3 (SP3) installed. You can download it from [here](https://support.microsoft.com/en-us/topic/description-of-forefront-unified-access-gateway-2010-service-pack-3-ce1d30a0-6d9a-6402-bbba-2b8aeefc76db)[^3^].
- A backup of your Forefront UAG configuration and data.
- A valid license key for Forefront UAG 2010.

## Installation steps
 
To install Forefront UAG 2010 SP4, follow these steps:

1. Download the Forefront UAG 2010 SP4 package from [here](https://support.microsoft.com/en-us/topic/description-of-forefront-unified-access-gateway-2010-service-pack-4-44cfd253-4648-a3b0-9b73-3976105248eb)[^1^]. The file name is **UAG\_KB2898607\_ENU.msp**.
2. Run the file as an administrator on each Forefront UAG server in your array.
3. Follow the instructions in the wizard to complete the installation.
4. Restart the server when prompted.
5. Repeat steps 2 to 4 for each server in your array.
6. Activate your Forefront UAG configuration.

## Verification
 
To verify that Forefront UAG 2010 SP4 is installed correctly, you can do the following:

- Check the version number of the Forefront UAG files in the **C:\Program Files\Microsoft Forefront Unified Access Gateway\bin** folder. The version number should be **4.0.1773.10100**.
- Check the version number of the Forefront UAG Management console in the **C:\Program Files\Microsoft Forefront Unified Access Gateway\Management\bin** folder. The version number should be **4.0.1773.10100**.
- Check the version number of the Forefront UAG Endpoint Components Installer in the **C:\Program Files\Microsoft Forefront Unified Access Gateway\von\InternalSite\PortalHomePage\Installers** folder. The version number should be **4.0.1773.10100**.
- Check the version number of the Forefront UAG Endpoint Components Installer on a client computer that connects to a Forefront UAG portal. The version number should be **4.0.1773.10100**.
- Test the functionality of your published applications and verify that they work as expected.

## Features and fixes in Forefront UAG 2010 SP4
 
Forefront UAG 2010 SP4 includes the following features and fixes:

- Added support for Windows 8.1 client devices that use Internet Explorer 11 and Internet Explorer 11 on the desktop. Users can also use the built-in Mail application on Windows 8.1 computers to connect to a published Exchange server, and the Remote Desktop Connection (RDC) 8.1 client to access a published Remote Desktop server.
- Added a tool (Export2tspub.exe) for publishing RemoteApp programs from a Remote Desktop Session Host (RD Session Host) on a server that is running Windows Server 2012 or Windows Server 2012 R2.
- Fixed various issues related to service stability, performance, authentication, application publishing, and DirectAccess connectivity.
